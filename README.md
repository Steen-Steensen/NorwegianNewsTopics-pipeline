# Norwegian News Corpus Analysis Pipeline

R pipeline for LDA topic modeling and matching social media posts (Instagram, Facebook, TikTok) 
to a corpus of Norwegian news articles, and assigning LDA topic distributions to
both the articles and the matched posts.

The pipeline was developed for the Algored project (Algorithmic editors and news diversity) at 
the Department of Communication, Kristiania University College, which analysed news diversity across
platforms in Norway (2023 corpus). The project was funded by the Norwegian Media Authority (2024-2025)

---

## Dataset deposit

The output of this pipeline — article metadata and topic distributions for
224,905 Norwegian news articles published in 2023, without the original news
texts — is publicly available on DataverseNO:

> Steen Steensen, 2026, *NorwegianNewsTopics (2023): Topic Modeling of Norwegian News*,
> <https://doi.org/10.18710/473JEF>, DataverseNO, V1

The deposit contains:

- `1_NorwegianNewsTopics.csv` — main dataset: one row per article, publication
  metadata and Topic1–Topic28 scores (pipeline output of script 14, minus
  body text and other original article content)
- `2_NorwegianNewsTopics_variable_dictionary.csv` — variable-level documentation
- `3_NorwegianNewsTopics_topic_dictionary.csv` — topic labels, descriptions and
  cluster assignments
- `4_Methods_LDA_and_SoMe_matching.pdf` — full methodological appendix
  describing the LDA topic modelling and social media matching procedures
  implemented in this pipeline

## Working directory convention

**All scripts assume that all input files and all scripts are in the same
folder, which is set as R's working directory.** No subfolder paths are used.

The simplest setup is to open the project in RStudio (via `Algored_final.Rproj`)
or to run `setwd("path/to/folder")` at the top of each session. Every
`readRDS()`, `read.csv()`, and `saveRDS()` call references files by filename
only, with no directory prefix.

## Prerequisites

### R packages

Install all required packages before running:

```r
install.packages(c(
  "dplyr", "stringr", "data.table", "pbapply", "readr",
  "tidytext", "tidyverse", "quanteda", "tm", "textstem", "stopwords",
  "Matrix", "text2vec", "purrr", "furrr", "progress",
  "mallet", "rJava", "segmented", "ggplot2",
  "reticulate", "cld3", "parallel",
  "httr", "jsonlite",
  "writexl", "openxlsx"
))
```

### Java and MALLET (required for script 04)

Script 04 runs LDA topic modeling via the MALLET toolkit, which requires a
Java Development Kit (JDK) and MALLET to be installed and available on the
system PATH.

- **macOS:** `brew install --cask temurin` (JDK), `brew install mallet`
- **Manual install:** Download MALLET from http://mallet.cs.umass.edu/download.php,
  extract it, and add the `bin/` directory to your PATH.
- Verify the installation by running `mallet import-file --help` in a terminal.
- Start a new R session after installing so that R inherits the updated PATH.

### Python environment (required for script 01)

Script 01 uses the fastText language model (via `reticulate`) to detect
Norwegian Nynorsk articles. You need a Python virtual environment with
`fasttext` and `huggingface_hub` installed:

```bash
python -m venv fasttext_env
source fasttext_env/bin/activate        # Windows: fasttext_env\Scripts\activate
pip install fasttext huggingface_hub
```

Then set the path to that Python executable in script 01 (see the `TODO`
comment near the `Sys.setenv(RETICULATE_PYTHON = ...)` line).

The Nynorsk classifier model (`nb-nbnn-lid.ftz`) is downloaded automatically
from HuggingFace on first run.

### OpenAI API key (required for script 02)

Script 02 translates Nynorsk articles to Bokmål using the OpenAI API
(`gpt-4o-mini`). Set your API key in the `openai_api_key` variable at the
top of the script.

## Pipeline overview

```
01  Preprocess articles                 → alltexts_original_pre_TP.rds
02  Translate Nynorsk, clean text       → alltexts_preprocessed.rds
03  Build document-term matrix          → dtm_lda_optimized.rds
04  LDA topic modeling (MALLET)         → doc-topics-28.txt, topic-keys-28.txt
05  Load LDA results                    → alltexts.rds, topic_distribution.rds
06  Add topic values to corpus          → alltexts_with_TP_distribution.rds
                                           topic_top_words.xlsx
 │
07  (Utility) Cosine similarity function — sourced by 08, 11, 12
 │
08  Match Instagram via cosine sim      → final_results_Insta.rds
                                           newsroom_results/ (TF-IDF cache, reused by 11 & 12)
09  Add topics to Instagram matches     → Insta_Data_Matched_TP.rds
 │
10  Match Facebook: article ID + URL    → FB_Data_Matched.rds
                                           FB_Data_filtered_remaining.rds
11  Match Facebook: cosine similarity   → final_results_FB.rds
12  Match TikTok: cosine similarity     → final_results_Tiktok.rds
13  Add topics to TikTok matches        → TikTok_Data_Matched_TP.rds
 │
14  Combine all matches into alltexts   → alltexts_final.rds
```

## Script reference

### 01 · `01_preprocess_articles.R`
Builds combined text fields, removes duplicates, filters articles outside 2023,
assigns unique IDs, detects non-Norwegian articles (cld3), and detects Norwegian
Nynorsk articles (fastText). **Requires Python environment** — see Prerequisites.

### 02 · `02_translate_nynorsk_clean.R`
Translates Nynorsk articles to Bokmål using the OpenAI API, then applies text
cleaning and lemmatisation to all articles. **Requires OpenAI API key.**

### 03 · `03_create_DTM.R`
Tokenises cleaned text, removes Norwegian stopwords, and builds a
Document-Term Matrix (DTM) optimised for LDA.

### 04 · `04_LDA_topic_modeling.R`
Trains LDA models with varying k values using MALLET, evaluates them with
log-likelihood and segmented regression, and trains the final model at k=28.
**Requires MALLET on PATH** — see Prerequisites.

### 05 · `05_load_LDA_results.R`
Loads MALLET output files and merges document-topic distributions into the
article dataset.

### 06 · `06_add_topic_values.R`
Merges topic distributions into the full article corpus, exports the top 50
articles per topic for manual validation, and exports the top 10 defining words
per topic for labelling.

### 07 · `07_cosine_similarity_function.R`
Utility script defining the `match_some_posts()` function used by scripts 08,
11, and 12. Not run directly — sourced by the matching scripts.

### 08 · `08_match_instagram.R`
Matches Instagram posts to articles using cosine similarity (TF-IDF via
text2vec), with a ±2-day date window. Precomputes per-newsroom TF-IDF matrices
and saves them to `newsroom_results/` for reuse by scripts 11 and 12.

### 09 · `09_match_instagram_topics.R`
Joins Instagram match results with article IDs and topic distributions.

### 10 · `10_match_facebook_url_id.R`
Matches Facebook posts to articles via two exact methods: (1) article ID
extraction from post URLs, and (2) direct URL hash join. Checks both `link`
and `final_link` columns. Passes unmatched posts to script 11.

### 11 · `11_match_facebook_cosine.R`
Matches remaining (unmatched) Facebook posts using cosine similarity, reusing
the TF-IDF cache from script 08.

### 12 · `12_match_tiktok.R`
Matches TikTok posts to articles using cosine similarity, reusing the TF-IDF
cache from script 08. Reads one CSV file per newsroom account.

### 13 · `13_match_tiktok_topics.R`
Joins TikTok match results with article IDs and topic distributions.

### 14 · `14_combine_social_media_matches.R`
Combines all social media match results and adds `published_on_fb`,
`published_on_insta`, and `published_on_tiktok` boolean columns to the full
article corpus. Similarity thresholds are set at the top of the script and
should be reviewed against sampled results from scripts 08–12.

## Notes

- The social media CSV files (`Instagram_Data.csv`, `Facebook_Data.csv`, TikTok
  CSVs) must have their newsroom identifier columns already matching the
  newsroom keys in the article corpus (lowercase, no spaces or special
  characters, e.g. `"tv2nyheter"`). Scripts 08, 10, 11, and 12 assume this
  and will produce no matches if the names differ.
- All scripts use `select <- dplyr::select` to resolve namespace conflicts
  between `dplyr` and `data.table`/`tm`.
- The `newsroom_results/` folder (TF-IDF cache) is created by script 08 and
  must exist before running scripts 11 or 12. If re-running from scratch,
  run script 08 first even if Instagram matching is not needed.
- Facebook matching uses two separate mechanisms: exact matches (script 10,
  no threshold) and cosine similarity (script 11, threshold applied). Script 14
  combines both.
